Import companies

Upload a CSV, map columns to Productboard fields, then run the import.
How to use
  • Upload — drop a CSV file (or click to browse), then map CSV columns to Productboard fields.
  • Auto-mapping — columns are auto-mapped when headers match common names (pb_id, name, domain, etc.).
  • Saved mappings — your mappings are saved to the browser and auto-restore on your next upload.
  • Options — configure multi-select mode and bypass settings before importing.
  • Validate first — use "Validate" to check for errors before running the import.
  • Live log — the log streams results in real time; download it afterwards for review.

Delete companies from CSV

Upload a CSV containing company UUIDs and permanently delete those companies.
How to use
  • Upload — drop a CSV containing company UUIDs (you can use an export CSV — just pick the ID column).
  • Select column — choose which column contains the UUIDs, then confirm deletion.
  • Irreversible — this action cannot be undone. Export your companies first if you need a backup.
⚠️ This action is irreversible. Deleted companies cannot be recovered. You can use the export CSV directly — just select the id column.

Source migration

Sync source identity fields between the legacy v1 API and v2 metadata.
How to use
  • V1 → V2 — copies source identity fields from the legacy API into v2 metadata.
  • V2 → V1 — copies v2 metadata back into legacy fields.
  • Independent — each direction runs separately. Use "Stop" to cancel mid-process.

V1 → V2 upcopy

Copies sourceOrigin and sourceRecordId from the legacy v1 API into v2 metadata (source.system / source.recordId) for every company that has v1 source data. Overwrites any existing v2 source metadata.
