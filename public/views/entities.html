Import entities

Upload CSV files for the entity types you want to import. Process multiple types in one run — entities are created in dependency order.
How to use
  • Upload — drop CSV files onto the entity type tiles (one file per type). You can import multiple types at once.
  • Map columns — template-format headers are auto-mapped. Adjust mappings per entity type using the tabs.
  • Options — configure multi-select mode, bypass settings, auto-generate ext_keys, and fiscal year start.
  • Validate first — use "Validate" to check for errors before running the import.
  • Dependency order — entities are created parent-first so relationships resolve correctly.
  • Live log — the log streams results in real time; download it afterwards for review.

Delete entities by CSV

Upload exported CSV files for each entity type you want to delete. Use the column picker on each tile to select the ID column. Entities are deleted in child-first order.
How to use
  • Upload — drop CSV files onto the entity type tiles (you can use export CSVs directly).
  • Pick ID column — select which column contains the entity UUIDs on each tile.
  • Safe mode — when enabled, entities with children not in your CSVs are skipped instead of cascade-deleted.
  • Child-first order — deletion runs from leaf entities up to parents, avoiding dependency errors.

Delete entities by type

Select entity types to permanently delete. Deleting a parent type cascade-deletes all its children automatically.
How to use
  • Select types — check the entity types you want to delete. Selecting a parent type (e.g. Product) auto-checks its children — they'll be cascade-deleted by Productboard.
  • Export first — there is no way to recover deleted entities. Export a backup before continuing.
  • Only parents are deleted via API — locked checkboxes indicate types that will be removed by cascade, not by direct API call.
  • Confirm — type DELETE and click the button to start the deletion.
⛔ This permanently deletes all entities of the selected types. This action cannot be undone. Export a backup before continuing.
