Merge Duplicate Companies Beta
Find and merge duplicate company records — consolidating notes and user associations into the canonical record before deleting duplicates.
How to use
  • Export first — download a full companies export before running. This is your only rollback option — use Companies → Export.
  • Select source — choose which source origin identifies the canonical (target) company, or pick Let me choose per group to manually select the target for each duplicate group.
  • Scan — click "Scan for duplicates" to detect duplicate company groups via the API. No changes are made during scan.
  • Preview — review the duplicate groups. Each group shows the target company and the duplicates that will be deleted. In manual mode, click any row to compare companies side-by-side and change the target. Select individual groups or process all.
  • Merge — for each duplicate company:
    1. All notes linked to the duplicate are found and relinked to the target company.
    2. If a note's customer is a user, the user's parent company is updated to the target instead.
    3. Any remaining users directly parented to the duplicate (with no notes) are also relinked to the target.
    4. The duplicate company is deleted — only after every relink for that company succeeds.
  • No rollback — deleted companies cannot be restored via the API. Export first.
  • Source detection — in automatic mode, the target company is identified by its source origin. Domains with zero or multiple matches for the selected origin are skipped. In manual mode, all domains with 2+ companies are included.
⚠️ Before scanning, export your companies data from Companies → Export. This is your only rollback option — deleted companies cannot be recovered via the API.

Merge companies from CSV Beta
Upload a CSV with target and duplicate company UUIDs — merge specific groups without running a full scan.
How to use
  • Prepare a CSV — one row per merge group. Each row needs a target company UUID (kept) and one or more duplicate UUIDs (comma-separated, will be deleted).
  • Map columns — select which CSV column contains the target UUID and which contains the duplicates list.
  • Validate — target column must have exactly 1 UUID per row; duplicates column must have 1+ comma-separated UUIDs. Errors block the preview.
  • Preview — review the merge groups and note/user counts before committing.
  • Merge — relinks all notes and users from each duplicate to its target company, then deletes the duplicates.
  • No rollback — export your companies from Companies → Export first.
⚠️ Before uploading, export your companies data from Companies → Export. Deleted companies cannot be recovered via the API.
