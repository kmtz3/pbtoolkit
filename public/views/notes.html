Export notes

Download all Productboard notes as a CSV file.
How to use
  • Filter — optionally narrow by date range or "past N days/weeks/months" before exporting.
  • Export — click "Export notes" to fetch all matching notes (v2 API + v1 source enrichment).
  • Stop — use the "Stop" button to cancel a running export.
  • Download — the CSV downloads automatically when complete; click "Download CSV" to grab it again.
ℹ️ Fetches notes via the v2 API (relationships included inline), then enriches any missing source origin/record ID from the v1 API. Large workspaces may take a few minutes.

Columns exported: PB Note ID, Type, Title, Content, Display URL, User Email, Company Domain, Owner Email, Creator Email, Tags, Source Origin, Source Record ID, Archived, Processed, Created At, Updated At, Linked Entities.

Import notes

Upload a CSV, map columns to Productboard note fields, then run the import.
How to use
  • Upload — drop a CSV file (or click to browse), then map CSV columns to note fields.
  • Saved mappings — your mappings are saved to the browser and auto-restore on your next upload.
  • Migration mode — enable this when importing into a different workspace to remap linked entity UUIDs via a custom text field.
  • Validate first — use "Validate" to check for errors before running the import.
  • Live log — the log streams results in real time; download it afterwards for review.
ℹ️ Notes are created via the v1 API (which supports linking by user email or company domain). Status fields (archived/processed) and creator/owner are applied via a v2 backfill step after creation.

Migration prep

Transform an export CSV to make notes ready for import into a new workspace.
How to use
  • Upload — drop an export CSV from the source workspace.
  • Source name — enter a migration source name (used as source_origin in the output).
  • Download — the prepared CSV clears pb_id and moves it to source_record_id, ready for import into the target workspace.
  • Next step — import the prepared CSV using the Import tab with Migration mode enabled.
ℹ️ Workflow:
  1. Export notes from your source workspace (Export notes tab).
  2. Upload that CSV here and enter the migration source name.
  3. Download the transformed CSV — each note's pb_id is moved to source_record_id (so it can be deduplicated on re-import), source_origin is set to your source name, and pb_id is cleared so Productboard creates a fresh note in the target workspace.
  4. Import the transformed CSV into the target workspace (Import notes tab). Enable Migration mode to also remap linked entity UUIDs.
