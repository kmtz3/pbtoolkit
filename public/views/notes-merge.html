Merge duplicate notes Beta
Find and merge duplicate notes — consolidating tags, product links, owners, and state into a single canonical note before deleting duplicates.
How to use
  • Export first — download a full notes export before running. This is your only rollback option — re-import via Notes → Import.
  • Configure — optionally scope by date range, choose which note to keep, and enable loose matching if needed.
  • Scan — click "Scan for duplicates" to detect groups. No changes are made during scan.
  • Preview — review the duplicate groups table carefully. Download the preview CSV for offline review.
  • Merge — confirm and run. Tags, product links, and owner followers are consolidated into the target; secondary notes are deleted.
  • Limitations — source data (origin / record ID) from secondary notes cannot be transferred and will be discarded. Restored notes receive new IDs. Note text body is never combined — only metadata (tags, links, owners, state) is consolidated into the target note's existing content.

How duplicates are detected

  • Matching key — two notes are considered duplicates when they share the same title, content, and customer (same company or same user). All three must match exactly (case-sensitive).
  • Loose match — when enabled, the title requirement is dropped. Notes with the same content and customer are grouped even if their titles differ. Use with caution and review the preview carefully.
  • Excluded notes — notes with no content or no customer relationship are excluded from detection entirely.

What happens during merge

  • Target note — the note you choose to keep (newest / oldest / most metadata). Its title, content, and source data are preserved unchanged.
  • Tags — all unique tags from secondary notes are added to the target. Already-present tags are skipped.
  • Product links — all product hierarchy links from secondary notes are added to the target. Already-linked entities are skipped.
  • State — the highest-priority state across the group is applied to the target. Priority order: Processed > Unprocessed > Archived.
  • Owners → followers — each secondary note's owner is added as a follower on the target note. The target's owner is not changed.
  • Customer relationship — if the target has only a company and a secondary has a specific user from that company, the user relationship is added to the target.
  • Source data — source origin and record ID from secondary notes are logged in the audit but cannot be transferred. They are shown in the preview table for reference.
  • Deletion — secondary notes are deleted only after all metadata consolidation steps succeed for that group.
⚠️ Before scanning, download a full notes export from Notes → Export. This is your only rollback option. Re-imported notes receive new IDs, source data cannot be restored, and the Productboard API does not support setting Created at — notes will be backdated only if re-imported via CSV (though this does not restore feature connections).
