Merge Duplicate Companies Beta
Relink notes and user associations from duplicate companies to their Salesforce canonical record, then delete the duplicates.
How to use
  • Generate the CSV — run the following SQL query against your Productboard database to produce the input file. Replace {space_id} with your workspace ID. 
    SELECT
  domain,
  array_agg(uuid || ' (' || source_origin || ')') AS uuid_with_origin
FROM organizations
WHERE space_id = {space_id} AND domain != ''
GROUP BY domain
HAVING COUNT(*) > 1
  AND bool_or(source_origin = 'salesforce')
    The query returns one row per domain that has more than one company record and at least one Salesforce-sourced record. Export the result as a CSV before uploading here.
  • CSV format — the file must have two columns: domain and uuid_with_origin. Each row represents one domain; uuid_with_origin lists all company UUIDs for that domain, each tagged with its source in parentheses (e.g. abc123… (salesforce), def456… (manual_or_csv)).
  • Exactly one Salesforce UUID per row — rows with zero or multiple (salesforce) UUIDs are skipped automatically and shown in the preview.
  • Dry run first — the default mode logs every action without making any changes. Review the log, then run live to execute.
  • What happens during a live run:
    1. All notes linked to each duplicate company are found via POST /v2/notes/search.
    2. Each note is relinked to the Salesforce company — if the note's customer is a user, the user's parent company is updated; if the customer is a company, the note's customer relationship is repointed.
    3. The duplicate company is deleted — only after every relink for that company succeeds. If any relink fails the delete is skipped.
  • No rollback — deleted companies cannot be restored via the API. Export your companies data first from Companies → Export.
⚠️ Export your companies data before running. Deleted companies cannot be recovered via the API. Use Companies → Export to save a backup first.
